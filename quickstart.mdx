## Prerequisites

Before you begin, ensure you have:

- A modern web browser like Chrome, Firefox, or Safari
- An email address for account verification
- Optional: A GitHub account for version control integration

<Callout kind="info">
  No credit card or payment required to get started. You receive full access to core features immediately after signup.
</Callout>

## Create Your Account

Follow these steps to register and log in to Ben Smith.

<Steps>
  <Step title="Visit the Dashboard" icon="monitor">
    Navigate to [https://dashboard.bensmith.com](https://dashboard.bensmith.com).

    Click the **Sign Up** button in the top-right corner.
  </Step>
  <Step title="Enter Your Details" icon="user">
    Provide your email address and create a strong password.

    Click **Create Account**.
  </Step>
  <Step title="Verify Email" icon="mail">
    Check your inbox for a verification email from Ben Smith.

    Click the verification link to activate your account.
  </Step>
  <Step title="Log In" icon="log-in">
    Return to the dashboard and sign in with your credentials.

    You now land on the main workspace dashboard.
  </Step>
</Steps>

## Create a New Documentation Space

Documentation spaces organize your project docs. Create one in seconds.

<Steps>
  <Step title="Click New Space" icon="plus">
    From the dashboard, select **New Documentation Space**.
  </Step>
  <Step title="Name Your Space" icon="edit-3">
    Enter a name like `My Project Docs` and add an optional description.

    Choose visibility: Public, Private, or Workspace.
  </Step>
  <Step title="Customize Settings" icon="settings">
    Set brand color (default: `#3B82F6`) and enable features like search or analytics.

    Click **Create Space**.
  </Step>
</Steps>

## Add and Edit Your First Document

Start authoring with our intuitive MDX editor.

<Tabs>
  <Tab title="Visual Editor" icon="edit">
    <Steps>
      <Step title="New Document">
        In your space, click **New Document** and name it `Introduction`.
      </Step>
      <Step title="Edit Content">
        Use the WYSIWYG editor to add headings, lists, and images.

        Preview changes in real-time.
      </Step>
      <Step title="Publish">
        Click **Publish** to make it live.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Markdown Mode" icon="code">
````markdown
# Welcome to My Docs

## Overview

This is your first document.

- Feature one
- Feature two
````
    Switch to Markdown tab, paste content, and preview.
  </Tab>
</Tabs>

<Callout kind="tip">
  Use `{variable}` syntax in backticks for code-like text in your docs. Escape `<` and `>` as `<` and `>` to prevent parsing issues.
</Callout>
